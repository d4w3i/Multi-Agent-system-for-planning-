Metrics Reference
Files F1▾
2 · P · R / (P + R) where P = matched / predicted, R = matched / actual
Harmonic mean of Precision and Recall over the set of files to modify. Matching is path-aware with 3 quality levels: exact path (3) > suffix/prefix match (2) > basename only (1). Greedy exclusive bipartite assignment — each file matches at most once.
Functions F1▾
2 · P · R / (P + R) where P = matched / predicted, R = matched / actual
Same bipartite F1 formula applied to function names. Matching uses 3 quality levels: fully-qualified name (3) > short name (2) > last component after stripping class prefix (1). Harder than file identification as function names must resolve to specific callables.
Semantic Score▾
0.30 · summary_sim + 0.70 · avg_step_sim
Weighted combination of summary similarity and step similarity. Steps are embedded with OpenAI text-embedding-3-large using operation + function + reason (file_to_modify excluded to avoid contamination). Steps are greedily matched by cosine similarity; unmatched predicted steps score 0.
Summary Similarity▾
cosine_sim(embed(summary_gt), embed(summary_pred))
Cosine similarity between OpenAI text-embedding-3-large embeddings of the ground-truth plan summary and the predicted plan summary. Captures high-level intent alignment. Also contributes 30% weight to the overall Semantic Score.
Step |Δ| (Abs Diff)▾
|predicted_steps − actual_steps|
Signed difference in step count. actual_steps is derived from GT structure: for each modified file, count its functions (or 1 if no functions listed). Positive = over-decomposition. Negative = under-prediction. σ measures variability across PRs, not run-to-run.
Target Coverage▾
matched_targets / total_predicted_steps
Fraction of predicted steps whose file or function target matches a ground-truth entry (path-aware / name-aware quality match). Measures how precisely the model's plan points at real targets — not how many GT steps are covered.
